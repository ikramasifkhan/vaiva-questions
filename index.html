Explain reverse routing in Laravel
  1. Laravel Reverse routing is the process of generating the URL, based on name or symbol.
  2. It generates URL's based on route declarations.
  3. Reverse routing makes your application so much more flexible and helps the developer to write cleaner codes in View.
  4. It defines a relationship between links and Laravel routes.
// route declaration
Route::get('login', 'users@login');
// reverse routing
{{ HTML::link_to_action('users@login') }}
